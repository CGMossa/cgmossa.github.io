This site is intentionally small.

I wanted a place that does not depend on an algorithm, a platform redesign,
or a pile of JavaScript to be useful. A static site suits me well: plain files,
predictable tooling, and something I can shape without asking anyone for permission.

I will use it for a few different kinds of writing:

- notes on Rust and R tooling
- thoughts on research software and maintainership
- modelling, statistics, and computational work from the scientific side
- short project updates when I ship something worth documenting

The site now runs on [Zola](https://www.getzola.org/), which feels like a good
fit for that goal: fast, boring, and easy to keep in version control.
